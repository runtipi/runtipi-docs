# Network Issues

Track down connectivity problems between users, proxy, and apps.

## Overview

- Time: 5–20 minutes
- Difficulty: Intermediate
- Prerequisites: Basic networking and DNS knowledge

## Symptoms

- Site doesn’t load, or loads intermittently
- SSL certificate/redirect problems
- App is reachable on localhost but not from the internet

## Checklist

- DNS points to the correct public IP
- Firewall allows 80/443 (and any needed custom ports)
- Reverse proxy routes to the correct service and port
- Container is healthy on the internal network

## Debug tips

- From the server: `curl -I https://your-domain.com`
- Inside the network: `docker exec -it <proxy> sh` and test connectivity
- Validate certificates and expiration
