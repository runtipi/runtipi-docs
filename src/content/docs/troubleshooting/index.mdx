## Quick Fixes

### Runtipi Won't Start

```bash
# Check if Docker is running
sudo systemctl status docker

# Restart Docker
sudo systemctl restart docker

# Try starting Runtipi
cd /path/to/runtipi
sudo ./runtipi-cli start
```

### App Won't Install

```bash
# Check logs for the specific app
sudo ./runtipi-cli logs app-name

# Try restarting the app
# (Stop and start from the dashboard)

# Check disk space
df -h

# Check if port is already in use
sudo netstat -tulpn | grep :PORT_NUMBER
```

### Can't Access Dashboard

**Check if Runtipi is running:**
```bash
sudo ./runtipi-cli status
```

**Check your firewall:**
```bash
sudo ufw status
# Make sure port 80/443 are allowed or your custom port
```

**Verify your IP address:**
```bash
hostname -I
```

### Dashboard is Slow

```bash
# Check system resources
htop  # or top

# Check Docker containers
docker ps
docker stats

# Restart Runtipi
sudo ./runtipi-cli restart
```

## Frequently Asked Questions (FAQ)

### General Questions

**Q: What are the minimum requirements to run Runtipi?**

A: You need a 64-bit processor, 4GB RAM minimum (8GB recommended), and 10GB free disk space. Runtipi supports x86_64 and ARM64 architectures.

---

**Q: Can I run Runtipi on a Raspberry Pi?**

A: Yes! Raspberry Pi 4 and 5 with ARM64 architecture are supported. Note that some apps may not be available on ARM.

---

**Q: Does Runtipi work on Windows or macOS?**

A: No, Runtipi requires a Linux server. You can run it in a Linux VM on Windows/macOS, but native support is not available.

---

**Q: How do I update Runtipi?**

A: Run `sudo ./runtipi-cli update latest` from your Runtipi directory.

---

**Q: Will updating Runtipi break my apps?**

A: Generally no, but check the [release notes](/docs/reference/release-notes) for breaking changes. Always backup first.

---

### Installation Questions

**Q: The installation script fails with "Docker not found"**

A: Make sure you have Docker installed. If you used `apt install docker.io`, uninstall it and use the official script:
```bash
curl -fsSL https://get.docker.com | sh
```

---

**Q: "Permission denied" errors during installation**

A: Make sure you're running with sudo:
```bash
sudo curl -L https://setup.runtipi.io | sudo bash
```

---

**Q: Can I install Runtipi in a custom directory?**

A: Yes, just cd to that directory before running the install script.

---

### App Questions

**Q: Why can't I find [specific app] in the App Store?**

A: The app might not be available yet, or it might not support your architecture (e.g., ARM). You can request new apps on [GitHub](https://github.com/runtipi/runtipi-appstore/issues).

---

**Q: Can I install apps that aren't in the App Store?**

A: Not directly through the dashboard, but you can create a [custom app store](/docs/guides/create-your-own-app-store) or run Docker containers manually alongside Runtipi.

---

**Q: Where is my app data stored?**

A: All app data is in `app-data/[app-name]/` in your Runtipi directory.

---

**Q: How do I backup my apps?**

A: See the [Backup & Restore guide](/docs/guides/backup-and-restore). The quick version: backup the `app-data/` directory.

---

**Q: An app is stuck on "Installing"**

A: Check logs with `sudo ./runtipi-cli logs app-name`. Common causes:
- Slow internet (container images are large)
- Port conflicts
- Insufficient disk space
- Docker daemon issues

---

### Networking Questions

**Q: Can't access apps from other devices on my network**

A: Make sure you're using your server's IP address (not `localhost`). Check your firewall isn't blocking connections.

---

**Q: Do I need a domain name to use Runtipi?**

A: No! You can use IP addresses. Domains are only needed for:
- SSL certificates (HTTPS)
- Exposing apps to the internet
- Prettier URLs

---

**Q: What's the difference between exposing an app and accessing it locally?**

A: "Exposing" makes it accessible from the internet with a domain name. Local access only works on your network with IP:port.

---

**Q: Can I use Runtipi behind another reverse proxy?**

A: Yes, but it's complex. Runtipi uses Traefik internally. You'd need to configure your external proxy to forward to Runtipi's Traefik.

---

### Security Questions

**Q: Is it safe to expose Runtipi to the internet?**

A: Only with proper security measures:
- Strong passwords (20+ characters)
- HTTPS/SSL enabled
- Firewall configured
- Regular updates
- Consider using Cloudflare Tunnel instead

---

**Q: Should I run Runtipi on a VPS?**

A: It's possible but requires extra security setup. See [VPS Security](/docs/security/vps-security). Home servers behind routers are generally more secure for beginners.

---

**Q: How do I enable two-factor authentication?**

A: Runtipi dashboard 2FA support is in development. For now, use apps that have built-in 2FA (like Nextcloud, Vaultwarden).

---

### Troubleshooting Questions

**Q: Runtipi uses too much disk space**

A: Check app data sizes:
```bash
du -sh app-data/*
```
Large apps (media servers, cloud storage) can use hundreds of GB. Clean up unused data or add more storage.

---

**Q: Runtipi is using too much RAM/CPU**

A: Check which apps are consuming resources:
```bash
docker stats
```
Some apps (media transcoding, databases) are resource-intensive. Consider upgrading hardware or stopping unused apps.

---

**Q: "Port already in use" error**

A: Another service is using that port. Find it:
```bash
sudo netstat -tulpn | grep :PORT
```
Either stop that service or change the app's port in settings.

---

**Q: Can't access Runtipi after reboot**

A: Runtipi should auto-start. If not:
```bash
cd /path/to/runtipi
sudo ./runtipi-cli start
```

To enable auto-start on boot, Runtipi sets this up during installation. If broken, reinstall or configure Docker's restart policies.

- Steps to reproduce the issue
- What you've already tried
