# Data & Storage in Runtipi

Understand where your app data lives and how to back it up safely.

## How Runtipi stores data

- Persistent data lives in the `app-data/` directory inside your Runtipi installation.
- Each app keeps its own data directory so updates don’t erase data.
- Configuration is stored alongside app definitions for easy portability.

## Backups: what to include

- App data directory (databases, uploads, configs)
- Any custom `.env` or configuration files (`user-config/`)
- Compose files (`apps/` or the Runtipi project directory) for reproducibility

## Observability and growth

- Monitor disk usage: `docker system df`, `du -sh <path>`
- Keep a little free space for updates and log growth
- Rotate logs if they grow too large
